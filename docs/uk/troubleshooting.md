# Усунення несправностей

## Дисплей показує чорний екран

**Симптоми:** Дисплей не засвічується після увімкнення.

**Можливі причини та рішення:**

1. **Неправильне підключення** — Перевірте кожен SPI-пін за таблицею підключення:
   - SCK → GPIO 4
   - MOSI → GPIO 6
   - DC → GPIO 7
   - CS → GPIO 10
   - RST → GPIO 5
2. **Проблема з живленням** — Переконайтеся, що VCC підключено до піна **3.3 В**, а не 5 В.
3. **Ненадійне з'єднання** — Перевірте всі дроти-перемички.
4. **Прошивка не запускається** — Відкрийте Thonny, підключіться до пристрою та перевірте REPL на наявність повідомлень про помилки.

---

## Енкодер не прокручує меню

**Симптоми:** Обертання енкодера не дає результату, або меню стрибає хаотично.

**Можливі причини та рішення:**

1. **Неправильні піни** — Перевірте: CLK → GPIO 3, DT → GPIO 2.
2. **Обернений напрямок** — Якщо меню прокручується у неправильному напрямку — поміняйте місцями дроти CLK і DT.
3. **Занадто великий поріг** — За замовчуванням `ENCODER_THRESHOLD = 2` вимагає 2 імпульси на крок. Якщо енкодер дуже чутливий — зменшіть до `1` у `src/main.py`.
4. **Незаземлені піни** — Усі піни енкодера використовують `Pin.PULL_UP`. Якщо спостерігається випадковий рух — перевірте, що GND енкодера правильно підключено.

## Кнопка не реагує

**Симптоми:** Натискання кнопки енкодера не дає результату.

**Можливі причини та рішення:**

1. **Неправильний пін** — Перевірте: SW → GPIO 1.
2. **Ненадійний дріт** — Перевірте з'єднання SW і GND.
3. **Затримка антидребезгу** — Після кліку прошивка чекає 120 мс перед прийняттям наступного сигналу. Це нормально.
4. **Поріг довгого натискання** — Натискання коротше 900 мс — короткий клік; довше — довгий клік. Переконайтеся, що не тримаєте кнопку занадто довго, коли очікуєте короткий клік.

---

## Портал Wi-Fi не відкривається

**Симптоми:** Після вибору Wi-Fi у меню мережа `BFU-SETUP` не з'являється на телефоні.

**Можливі причини та рішення:**

1. **Зачекайте** — Точка доступу запускається 1–2 секунди. Зачекайте і оновіть список Wi-Fi на телефоні.
2. **AP вже активна** — Якщо портал був запущений раніше і не зупинений — натисніть кнопку для повернення назад і знову увійдіть у меню Wi-Fi.
3. **Помилка сокета** — Перевірте REPL Thonny на наявність `Portal start error`. Це може вказувати на конфлікт портів. Скиньте пристрій і спробуйте знову.
4. **Телефон автоматично відключається** — Деякі телефони відключаються від мереж без інтернету. Натисніть **Залишитися підключеним** при появі запиту.

## Пристрій не підключається автоматично після перезавантаження

**Симптоми:** Після перезавантаження пристрій не підключається до Wi-Fi автоматично.

**Можливі причини та рішення:**

1. **Ще не налаштовано** — Якщо ви ще не підключалися через портал BFU-SETUP, файл `wifi_config.json` не існує. Підключіться вперше через **НАЛАШТУВАННЯ → WI-FI**.
2. **Мережа недоступна** — Якщо роутер вимкнений або поза зоною дії, пристрій завершить спробу підключення через ~15 секунд і запустить меню. Це нормальна поведінка.
3. **Пароль змінено** — Якщо пароль Wi-Fi змінився, перейдіть до **НАЛАШТУВАННЯ → WI-FI** і підключіться знову. Нові дані перезапишуть старий `wifi_config.json`.
4. **Пошкоджений файл конфігурації** — Якщо `wifi_config.json` пошкоджений, пристрій все одно запуститься. Підключіться знову через портал для відновлення файлу.

## Помилка YouTube API

**Симптоми:** Екран підписників показує `API ПОМИЛКА`.

**Можливі причини та рішення:**

1. **Немає Wi-Fi** — Пристрій повинен бути підключений до Wi-Fi. Перевірте екран статусу.
2. **Неправильний API-ключ** — Перевірте, що `YOUTUBE_API_KEY` у `src/main.py` — ваш реальний ключ, а не заглушка.
3. **API не увімкнено** — Перейдіть до Google Cloud Console і підтвердіть, що **YouTube Data API v3** увімкнено для вашого проєкту.
4. **Перевищено квоту** — Безкоштовна квота — 10 000 одиниць/день. Кожен запит підписників використовує 1 одиницю. Перевищення малоймовірне при звичайному використанні.
5. **Неправильний Channel ID** — Перевірте правильність `YOUTUBE_CHANNEL_ID`. Він повинен починатися з `UC`.
6. **Тайм-аут мережі** — Запит не має явного тайм-ауту. Якщо мережа повільна — він може зависнути. Скиньте пристрій і спробуйте знову.

## Кількість підписників відрізняється від YouTube Studio

**Симптоми:** Відображувана кількість менша або відрізняється від того, що ви бачите в YouTube Studio.

**Пояснення:**

Це очікувана поведінка. YouTube Data API v3 повертає **публічну кількість підписників**, яка:

- **Округлена** до 3 значущих цифр для каналів з більш ніж 1 000 підписників
- **Кешована** — може відставати від реального значення на хвилини або години

YouTube Studio показує точну кількість, яка доступна лише власнику каналу через автентифікований API. Публічний API, що використовується в цьому проєкті, не підтримує автентифікацію.

Це обмеження платформи YouTube і не може бути виправлено в прошивці.

## Помилка пам'яті при запуску

**Симптоми:** Пристрій аварійно завершує роботу з `MemoryError` або `OSError: [Errno 12] ENOMEM`.

**Можливі причини та рішення:**

1. **Застаріла прошивка** — Прошийте останню версію MicroPython для ESP32-C3 (дивіться [installation.md](installation.md)).
2. **Великий словник шрифтів** — Словник `FONT_UA` використовує значну частину RAM. Виклик `gc.collect()` перед великими операціями вже є у `fetch_youtube_subscribers()`.
3. **Фрагментація купи** — Скиньте пристрій. Якщо помилка зберігається після свіжого завантаження — перепрошийте прошивку.

## Неправильні номери пінів / Нічого не працює

**Симптоми:** Кілька компонентів не працюють після збірки.

**Контрольний список:**

- [ ] Ви використовуєте саме **ESP32-C3 Super Mini**? Інші варіанти ESP32 мають інше розташування пінів.
- [ ] Дисплей підключено до **SPI шини 1** (GPIO 4 = SCK, GPIO 6 = MOSI)?
- [ ] VCC підключено до **3.3 В**, а не 5 В?
- [ ] Усі з'єднання GND виконані?
- [ ] REPL Thonny показує повідомлення про помилки при запуску?

Якщо нічого не допомагає — відкрийте Thonny, підключіться до пристрою та виконайте:
```python
from machine import Pin
p = Pin(4, Pin.OUT)
p.value(1)
```
Це перевіряє, чи реагує GPIO 4. Якщо ні — плата може бути пошкоджена.
