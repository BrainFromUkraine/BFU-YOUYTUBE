# BFU YouTube Лічильник Підписників

> **ESP32-C3 Super Mini** · **Круглий дисплей GC9A01** · **Ротаційний енкодер** · **MicroPython**

Компактний автономний лічильник підписників YouTube на базі ESP32-C3 Super Mini. Відображає кількість підписників у реальному часі на круглому дисплеї GC9A01 240×240, дозволяє навігацію через меню за допомогою ротаційного енкодера та підключається до Wi-Fi через вбудований браузерний портал налаштування — без комп'ютера після першого прошивання.

---

## Можливості

- 🟢 **Кількість підписників у реальному часі** — отримує дані з YouTube Data API v3 кожні 60 секунд
- 📡 **Портал налаштування Wi-Fi** — SoftAP + веб-форма на `192.168.4.1`, без додатків
- 📷 **Екран з QR-кодом** — скануйте телефоном для швидкого підключення до точки доступу
- 🎛️ **Навігація через ротаційний енкодер** — прокрутка меню, короткий клік — вибір, довгий клік — назад
- 🖥️ **Круглий дисплей GC9A01** — 240×240 пікселів, SPI, повнокольоровий RGB565
- 🇺🇦 **Власний бітмап-шрифт** — підтримує кирилицю, латиницю та цифри
- 📊 **Екран статусу** — показує стан Wi-Fi та апаратного забезпечення
- ⚡ **Один файл MicroPython** — зовнішні бібліотеки не потрібні

## Підключення

### Дисплей GC9A01

| Пін дисплея | GPIO ESP32-C3 |
|---|---|
| SCK | GPIO 4 |
| MOSI | GPIO 6 |
| DC | GPIO 7 |
| CS | GPIO 10 |
| RST | GPIO 5 |
| VCC | 3.3 В |
| GND | GND |

### Ротаційний енкодер

| Пін енкодера | GPIO ESP32-C3 |
|---|---|
| CLK | GPIO 3 |
| DT | GPIO 2 |
| SW | GPIO 1 |
| VCC | 3.3 В |
| GND | GND |

> ⚠️ **Важливо:** ESP32-C3 працює на **логіці 3.3 В**. Не підключайте жоден пін безпосередньо до 5 В.

## Як це працює

1. **Завантаження** — пристрій ініціалізує дисплей і показує головне меню.
2. **Налаштування Wi-Fi** — перейдіть до *Налаштування → Wi-Fi*. Пристрій створює точку доступу `BFU-SETUP`. Підключіть телефон до неї, відкрийте `http://192.168.4.1`, виберіть домашню мережу та введіть пароль.
3. **Кількість підписників** — перейдіть до *Підписники*. Пристрій звертається до YouTube Data API і відображає поточну кількість. Оновлення відбувається автоматично кожні 60 секунд.
4. **Екран статусу** — показує модель ESP32-C3, стан дисплея, енкодера та Wi-Fi.

## Портал налаштування Wi-Fi

| Крок | Дія |
|---|---|
| 1 | Перейдіть до **Налаштування → Wi-Fi** на пристрої |
| 2 | Дисплей показує QR-код — відскануйте його камерою телефона |
| 3 | Телефон підключається до мережі `BFU-SETUP` (пароль: `12345678`) |
| 4 | Відкрийте браузер і перейдіть на `http://192.168.4.1` |
| 5 | Виберіть домашню Wi-Fi мережу зі списку |
| 6 | Введіть пароль Wi-Fi і натисніть **Connect** |
| 7 | Пристрій підключається і показує призначену IP-адресу |

## Налаштування YouTube API

1. Перейдіть до [Google Cloud Console](https://console.cloud.google.com/)
2. Створіть новий проєкт (наприклад, `BFU-Counter`)
3. Перейдіть до **APIs & Services → Library**
4. Знайдіть **YouTube Data API v3** і натисніть **Enable**
5. Перейдіть до **APIs & Services → Credentials**
6. Натисніть **Create Credentials → API key**
7. Скопіюйте згенерований ключ
8. Відкрийте `src/main.py` і замініть заглушку:
   ```python
   YOUTUBE_API_KEY = "PASTE_YOUR_API_KEY_HERE"
   ```
   на ваш реальний ключ.

> ⚠️ **Безпека:** Ніколи не публікуйте реальний API-ключ у публічному репозиторії. Файл `.gitignore` вже виключає `config.py` та `secrets.py`.

## Завантаження на ESP32-C3 через Thonny

1. Завантажте та встановіть [Thonny IDE](https://thonny.org/)
2. Прошийте MicroPython на ESP32-C3:
   - Завантажте останню прошивку ESP32-C3 з [micropython.org/download](https://micropython.org/download/esp32c3/)
   - У Thonny: **Tools → Options → Interpreter** → виберіть *MicroPython (ESP32)* та правильний COM-порт
   - Натисніть **Install or update MicroPython** і дотримуйтесь інструкцій
3. Відкрийте `src/main.py` у Thonny
4. Вставте ваш YouTube API-ключ у змінну `YOUTUBE_API_KEY`
5. Натисніть **File → Save as…** → виберіть *MicroPython device* → збережіть як `main.py`
6. Натисніть кнопку **Reset** на ESP32-C3 або зелену кнопку **Run** у Thonny

## Архітектура коду

| Модуль | Опис |
|---|---|
| Клас `GC9A01` | Повний SPI-драйвер дисплея — послідовність ініціалізації, `fill_rect`, `write_block`, `text`, `ua_text` |
| Словник `FONT_UA` | Власний 5×7 бітмап-шрифт з підтримкою української кирилиці, латиниці, цифр і символів |
| Ротаційний енкодер | Опитування з налаштовуваним порогом; визначає напрямок і короткий/довгий натиск кнопки |
| Система меню | Машина станів зі змінною `screen_mode`; підтримує вкладені меню та сторінки |
| YouTube API | `fetch_youtube_subscribers()` — HTTP GET до YouTube Data API v3, парсинг JSON |
| Wi-Fi портал | `start_wifi_setup_portal()` — SoftAP + неблокуючий TCP-сокет-сервер на порту 80 |
| Екран QR-коду | Попередньо обчислена матриця QR, відрендерена піксель за пікселем через `fill_rect` |
| Головний цикл | Один цикл `while True` — обробляє портал, енкодер, кнопку та періодичне оновлення API |

## Усунення несправностей

| Проблема | Рішення |
|---|---|
| Дисплей показує чорний екран | Перевірте підключення SPI (SCK, MOSI, DC, CS, RST). Перевірте живлення 3.3 В. |
| Енкодер не прокручує меню | Перевірте піни CLK та DT. Спробуйте поміняти їх місцями, якщо напрямок обернений. |
| Кнопка не реагує | Перевірте підключення піна SW. Переконайтеся, що `PULL_UP` активний. |
| Портал Wi-Fi не з'являється | Переконайтеся, що ви вибрали *Налаштування → Wi-Fi* і зачекали 1–2 секунди. |
| Помилка YouTube API | Перевірте правильність API-ключа та чи увімкнено YouTube Data API v3. |
| Кількість підписників відрізняється від YouTube Studio | Публічний API повертає округлене/кешоване значення. Це обмеження YouTube. |
| Помилка пам'яті при запуску | Переконайтеся, що використовуєте останню прошивку MicroPython для ESP32-C3. |
| Неправильні номери пінів | Перевірте таблицю підключення вище. Всі GPIO вказані для ESP32-C3 Super Mini. |
