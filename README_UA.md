# BFU YouTube Лічильник Підписників

> **ESP32-C3 Super Mini** · **Круглий дисплей GC9A01** · **Ротаційний енкодер** · **MicroPython**

Компактний автономний лічильник підписників YouTube на базі ESP32-C3 Super Mini. Відображає кількість підписників у реальному часі на круглому дисплеї GC9A01 240×240, дозволяє навігацію через меню за допомогою ротаційного енкодера та підключається до Wi-Fi через вбудований браузерний портал — після першого прошивання більше не потрібен USB-кабель або комп'ютер.

---

## Можливості

- 🟢 **Кількість підписників у реальному часі** — отримує дані через Railway backend кожні 60 секунд
- 🖼️ **Аватар каналу** — відображає аватар 32×32 у форматі бінарного RGB565 (без JSON, безпечно для пам'яті)
- 📡 **Портал налаштування Wi-Fi** — SoftAP + веб-форма на `192.168.4.1`, без додатків
- 📷 **Екран з QR-кодом** — скануйте телефоном для підключення до точки доступу
- 🎛️ **Навігація ротаційним енкодером** — прокрутка меню, короткий клік — вибір, довгий — назад
- 🖥️ **Круглий дисплей GC9A01** — 240×240 пікселів, SPI, повнокольоровий RGB565
- 🇺🇦 **Власний шрифт для кирилиці** — підтримує українські літери, латиницю та цифри, без `framebuf`
- 📊 **Екран статусу** — показує стан Wi-Fi та апаратного забезпечення
- ⚡ **Один файл MicroPython** — не потребує зовнішніх бібліотек на пристрої
- 🔒 **Railway backend обов'язковий** — API-ключ ніколи не зберігається на ESP32

## Підключення

### Дисплей GC9A01

| Пін дисплея | GPIO ESP32-C3 |
|---|---|
| SCK | GPIO 4 |
| MOSI | GPIO 6 |
| DC | GPIO 7 |
| CS | GPIO 10 |
| RST | GPIO 5 |
| VCC | 3.3 В |
| GND | GND |

### Ротаційний енкодер

| Пін енкодера | GPIO ESP32-C3 |
|---|---|
| CLK | GPIO 3 |
| DT | GPIO 2 |
| SW | GPIO 1 |
| VCC | 3.3 В |
| GND | GND |

> ⚠️ **Важливо:** ESP32-C3 працює на **логіці 3.3 В**. Не підключайте жоден пін безпосередньо до 5 В.

## Архітектура

```
ESP32-C3  ──►  Railway Backend  ──►  YouTube Data API v3
                    │
                    └──►  /api/subscribers   (JSON, кількість підписників)
                    └──►  /api/avatar-rgb565 (бінарний RGB565, 2048 байт)
```

Railway backend виступає захищеним проксі:
- Ваш YouTube API-ключ зберігається **лише** у змінних Railway — ніколи на ESP32.
- Backend кешує відповіді (за замовчуванням 30 с) для мінімізації використання квоти YouTube API.
- ESP32 робить два HTTP-запити на екрані підписників: один для кількості, один для аватара.

## Як це працює

1. **Завантаження** — пристрій ініціалізує дисплей, скидає Wi-Fi інтерфейс і намагається підключитися до збереженої мережі.
2. **Налаштування Wi-Fi** — перейдіть до *Налаштування → Wi-Fi*. Пристрій створює точку доступу `BFU-SETUP`. Підключіть телефон, відкрийте `http://192.168.4.1`, виберіть домашню мережу та введіть пароль. Дані зберігаються у `wifi_config.json` на пристрої.
3. **Кількість підписників** — перейдіть до *Підписники*. Пристрій звертається до Railway backend і відображає поточну кількість. Оновлюється автоматично кожні 60 секунд.
4. **Аватар каналу** — після завантаження кількості підписників пристрій отримує зображення 32×32 у форматі бінарного RGB565 (2048 байт) і відображає його через `display.write_block()`. Без парсингу JSON, без циклу конвертації — безпечно для пам'яті ESP32-C3.
5. **Екран статусу** — показує модель ESP32-C3, стан дисплея, енкодера та Wi-Fi.

## Портал налаштування Wi-Fi

| Крок | Дія |
|---|---|
| 1 | Перейдіть до **Налаштування → Wi-Fi** на пристрої |
| 2 | Дисплей показує QR-код — скануйте його камерою телефона |
| 3 | Телефон підключається до мережі `BFU-SETUP` (пароль: `12345678`) |
| 4 | Відкрийте браузер і перейдіть на `http://192.168.4.1` |
| 5 | Виберіть домашню Wi-Fi мережу зі списку |
| 6 | Введіть пароль Wi-Fi і натисніть **Connect** |
| 7 | Пристрій підключається і показує призначену IP-адресу |

## Налаштування Railway Backend

Railway backend є **обов'язковим** — прошивка не підтримує прямі виклики YouTube API.

Дивіться **[docs/uk/backend.md](docs/uk/backend.md)** для повного посібника з розгортання.

```
ESP32  ──►  https://your-app.up.railway.app/api/subscribers
ESP32  ──►  https://your-app.up.railway.app/api/avatar-rgb565
```

Після розгортання відкрийте `src/main.py` і встановіть:

```python
BACKEND_SUBSCRIBERS_URL = "https://YOUR-APP-NAME.up.railway.app/api/subscribers"
BACKEND_AVATAR_URL      = "https://YOUR-APP-NAME.up.railway.app/api/avatar-rgb565"
DEVICE_API_TOKEN        = "ваш_опціональний_токен"
```

> ℹ️ **Примітка:** YouTube Data API v3 повертає **публічну кількість підписників**, яку YouTube округлює для каналів з більш ніж 1 000 підписників. Це обмеження платформи YouTube.

## Завантаження на ESP32-C3 через Thonny

1. Завантажте та встановіть [Thonny IDE](https://thonny.org/)
2. Прошийте MicroPython на ESP32-C3:
   - Завантажте останню прошивку ESP32-C3 з [micropython.org/download](https://micropython.org/download/esp32c3/)
   - У Thonny: **Tools → Options → Interpreter** → виберіть *MicroPython (ESP32)* і правильний COM-порт
   - Натисніть **Install or update MicroPython** і дотримуйтесь інструкцій
3. Відкрийте `src/main.py` у Thonny
4. Встановіть `BACKEND_SUBSCRIBERS_URL` та `BACKEND_AVATAR_URL` на URL вашого Railway застосунку
5. Натисніть **File → Save as…** → виберіть *MicroPython device* → збережіть як `main.py`
6. Натисніть кнопку **Reset** на ESP32-C3

## Архітектура коду

| Модуль | Опис |
|---|---|
| Клас `GC9A01` | Повний SPI-драйвер дисплея — ініціалізація, `fill_rect`, `write_block`, `ua_text` |
| Словник `FONT_UA` | Власний бітмап-шрифт 5×7 — українська кирилиця, латиниця, цифри, символи. Без `framebuf`. |
| `fetch_youtube_subscribers()` | HTTP GET до Railway backend `/api/subscribers`, JSON-відповідь |
| `fetch_channel_avatar()` | HTTP GET до Railway backend `/api/avatar-rgb565`, бінарна відповідь (2048 байт), прямий `write_block()` |
| `_reset_sta()` | Вимикає AP і скидає STA перед підключенням — запобігає помилці Wi-Fi 0x0102 |
| Ротаційний енкодер | Опитування з налаштованим порогом; визначає напрямок і короткий/довгий натиск |
| Система меню | Автомат станів зі змінною `screen_mode`; підтримує вкладені меню та сторінки |
| Wi-Fi портал | `start_wifi_setup_portal()` — SoftAP + неблокуючий TCP-сокет-сервер на порту 80 |
| Екран QR-коду | Попередньо обчислена матриця QR, відображається піксель за пікселем через `fill_rect` |
| Головний цикл | Один цикл `while True` — обробляє портал, енкодер, кнопку та оновлення API |

### Оптимізація пам'яті (ESP32-C3)

- Модуль `framebuf` видалено — весь текст відображається через `ua_text()` з прямим записом пікселів
- Аватар передається як бінарний RGB565 (2048 байт) — без парсингу JSON, без списку пікселів, без циклу `int(px, 16)`
- Буфер пікселів звільняється через `del buf; gc.collect()` одразу після `write_block()`
- Ручне введення пароля Wi-Fi видалено — замінено веб-порталом (економить ~150 рядків і RAM)
- Прямий виклик YouTube API видалено — Railway backend є єдиним шляхом отримання даних

## Усунення несправностей

| Проблема | Рішення |
|---|---|
| Дисплей показує чорний екран | Перевірте SPI-підключення (SCK, MOSI, DC, CS, RST). Перевірте живлення 3.3 В. |
| Енкодер не прокручує | Перевірте піни CLK та DT. Спробуйте поміняти їх місцями, якщо напрямок неправильний. |
| Кнопка не реагує | Перевірте підключення піна SW. Переконайтеся, що `PULL_UP` активний. |
| Портал Wi-Fi не з'являється | Перейдіть до *Налаштування → Wi-Fi* і зачекайте 1–2 секунди. |
| Статус "НЕМА API" | `BACKEND_SUBSCRIBERS_URL` порожній — встановіть URL вашого Railway застосунку. |
| Статус "API ПОМИЛКА" | Railway backend недоступний або повернув помилку. Перевірте логи Railway. |
| Статус "НЕМА WI-FI" | Пристрій не підключений до Wi-Fi. Використайте портал для налаштування. |
| Помилка Wi-Fi 0x0102 | Виправлено в прошивці — `_reset_sta()` вимикає AP і скидає STA перед підключенням. |
| Аватар не відображається | Перевірте `BACKEND_AVATAR_URL`. Backend повинен повертати рівно 2048 байт. |
| Кількість підписників відрізняється від YouTube Studio | Публічний API повертає округлене/кешоване значення. Це обмеження YouTube. |
| Помилка пам'яті при запуску | Переконайтеся, що використовуєте останню прошивку MicroPython для ESP32-C3. |

## Безпека

- Ваш YouTube API-ключ зберігається **лише** у змінних Railway — ніколи в прошивці або на пристрої.
- `wifi_config.json` зберігає пароль Wi-Fi у відкритому вигляді на файловій системі ESP32. Не комітьте його в репозиторій. Він вже вказаний у `.gitignore`.
- `DEVICE_API_TOKEN` є опціональним, але рекомендованим для виробничих розгортань.
